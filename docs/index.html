How It Works

When you choose a profile, GitSwitch runs one pipeline (after verifying the SSH private key file exists):

  1. Git identity — git config --global for user.name and user.email
  2. HTTPS → SSH — sets url.git@github.com:.insteadOf to https://github.com/ so HTTPS remotes use SSH
  3. GitHub CLI — if gh is installed, gh auth switch --user for that profile’s username
  4. SSH config — updates the Host github.com block so IdentityFile is the profile’s key
  5. Agent — ssh-add -D then ssh-add on the new key

Profiles are saved in app preferences. On first launch, example profiles are created for you to edit in Settings. At startup, the app infers the active profile from Git config and the current GitHub IdentityFile.
